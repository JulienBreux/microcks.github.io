GETTING STARTED

Open source Kubernetes Native tool for API Mocking and Testing

TURN YOUR API CONTRACT, COLLECTION OR SOAP UI PROJECT INTO LIVE MOCKS IN SECONDS.

Business friendly

Ease collaboration and reuse

Use the tool your business expert knows: Postman, SoapUI, OpenAPI or whatever! No need to learn a new DSL or configuration syntax.

Experiment

Prototype fast, succeed or fail fast

Experiment new APIs using the Backend as a Service feature, play and iterate before creating your API contract.

REST and SOAP

Same tool for API and WebServices

Sure we do love new REST APIs but we may also have a bunch of legacy services. Why not applyging same practices? We support both!

Continuous Integeration

Mock and test continuously

Integrate seemlessly in your continuous build or pipelines. We provide plugins, CLI or containers for you to choose the best option.

Kubernetes native

Want the choice on deployment?

Microcks can be deployed on any cloud provider on in-house infrastructure using Kube. It is using Operator for full-automated maintenance and cruise control.

Scalable

Mocks do not have to be expensive!

Scale to hundreds of mocks, billions of hits on a single instance. Microcks is totally elastic thanks to cloud-native design.
USE YOUR SPECIFICATION FOR TESTING CONTINUOUSLY!. INTEGRATE MICROCKS FROM YOUR JENKINS, GITLAB OR TEKTON PIPELINE.

Install & run everywhere.

Microcks is container based and works on Kubernetes, OpenShift, Docker Compose or Nomad.

We provide an Operator on OperatorHub.io for easy and intelligent installation.

Kubernetes using Helm chart

$ helm install ./microcks --name microcks

OpenShift using Template

$ oc new-app --template=microcks-persistent

Docker Compose

$ docker-compose -f microcks.yml up -d

See the Getting Started Guide for other methods.
